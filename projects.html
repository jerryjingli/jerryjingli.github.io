Stack Overflow has become a tremendous knowledge repository for developers' thoughts and practices. Hyperlinks in Q&A discussions are an important source of information to understand Q&A discussions and enrich learning experience. When reading a post, a desirable goal would be to easily discover and access online resources relevant to the information (e..g, a hyperlink) in the post, as if these online resources were embedded in the post. Tens of millions of hyperlinks are disseminated in Stack Overflow, while wisdom on what web resources are salient is implicit in the millions of discussion threads.

we present the LinkLive framework to discover and recommend salient web resources to a developer who is interested in the information relevant to a hyperlink in a web page.



scvRipper: A Tool for scraping time-series interaction data from screen-captured videos

scvRipper is a computer-vision-based video scraping technique to recognize window-based applications in screen-captured videos, and to extract application-specific content from the recognized application windows (e.g., code fragments in file editor, error messages in IDE console output, URLs in URL bar of web browser, and keywords in search box of search engine). scvRipper tool takes as input a screen-captured video, i.e., a time-series stream of screenshots taken by screen-capture tools such as Snagit. It produces as output a time-series interaction data (i.e., software usage and application content) reverse-engineered from the video.
